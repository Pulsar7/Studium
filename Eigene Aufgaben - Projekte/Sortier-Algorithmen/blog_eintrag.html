Einleitung

Das heutige Projekt beschäftigt sich mit Sortieralgorithmen und deren Effizienz.

Bubble-Sort-Algorithmus

Das Prinzip sehr leicht zu verstehen und es ist auch sehr schnell zu erkennen wie ineffizient dieser Sortieralgorithmus ist. Gegeben sind beispielsweise 3 Zahlen in einer eher ungeordneten Reihenfolge: 7, 5, 3. Um nun die Zahlen von der Größe nach zu sortieren, geht der Algorithmus folgendermaßen vor:

  • 7, 5, 3
  • 5, 7, 3
  • 5, 3, 7
  • 3, 5, 7

Es wird beispielsweise die Zahl 7 ausgewählt und geprüft, ob sie größer als die nächste Zahl - in diesem Fall die 5 ist. Dann geht es immer so weiter bis die richtige Reihenfolge erreicht wurde. Wie oben schon erwähnt wurde, ist dieser Algorithmus sehr ineffizient für längere Zahlenreihen. Natürlich kann ein Computer dieses simple Prinzip schnell durchführen. Je größer die Zahlenreihen jedoch werden, kann man auch bei der Rechendauer einen exponentiellen Anstieg erkennen. Als Beispiel habe ich ein C-Programm geschrieben: 

#include <stdio.h>
#include <stdlib.h>
#include <time.h>

Die Header-Datei time.h wird benötigt um später pseudorandomisiert Zahlen in einem bestimmten Intervall auszuwählen. Damit nicht bei jedem Programmstart die gleichen Zahlen ausgewählt werden, wird die Funktion srand() benutzt: 

int main(int argc, const char **argv) {
    srand(time(NULL));
    int counter = 0, array[MAX_INTEGERS] = {0};
    for (int r = 0; r < MAX_INTEGERS; r++) {
        for (int a = 0; a <= MAX_INTEGERS; a++) {
            array[a] = pseud_rand();
        }
        sort_array(array,counter);
        counter++;
    }
    return 0;
}

Die ausgewählten Zahlen sollen in ein integer-Array gespeichert werden, dessen Größe zu Beginn definiert wurde: #define MAX_INTEGERS 10000. Als Beispiel habe ich hier 10000 ausgewählt.

Damit auch nicht immer die gleichen Zahlenfolgen vom Computer sortiert werden, werden alle Zahlen im Array jedes Mal neu angeordnet, bzw. "generiert". Dabei habe ich die pseud_rand()-Funktion sehr kurz gehalten: 
int pseud_rand(void) {
    return ((rand()%10000000)+0); // pseudorandom
}
